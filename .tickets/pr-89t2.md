# Rename slash commands to move namespace

Replace the current /relocate* command surface with the unpublished /move command surface. No compatibility aliases are needed before publish.

## Design

Register only the new public commands: /move <target>, /move-status, /move-lineage, and /move-prune. Rename command descriptions and usage text accordingly. Remove /relocate, /relocate-status, /relocate-lineage, and /relocate-prune registrations instead of aliasing them.

## Acceptance Criteria

- /move <target> performs the current session move formerly handled by /relocate.
- /move-status shows session move status.
- /move-lineage shows/names lineage, including --name support.
- /move-prune keeps the existing safe prune behavior under the new command name.
- No /relocate* compatibility aliases remain.
- npx tsc --noEmit passes.
